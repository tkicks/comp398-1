Classes

Classes are multiple use selectors

meaning that they can apply to as many different elements as you want

You can use them to style specific elements in your code without changing all elements of that type

Here is some example code in CSS

.stuff{font-family: helvetica, serif;}

Here is some example code in HTML

(td class="stuff")Nutrition Facts(/td)

ID's

What are they?

ID's are essentially a one-time-use version of classes.

When do you use them?

When there is one specific element (such as a paragraph or ordered list) that you want to apply special formatting to.

How do you use them?

Define them in the .css:

#idName{style rules}

-or-

elementType#idName{style rules}

Assign id to an element in the .html:

< elementType id="idName" ...>

Example:

In your .css:

p#phoneNum{

font-size:50px;
font-family:georgia,serif;
font-weight: bold;
font-style: oblique;
}

In your .html:

< p id=phoneNum>(123)456-7890< /p >

How it looks on the page:

(123) 456 - 7890

Pseudo-Selectors

Link Pseudo-Selectors

  • :link
  • :visited
  • :hover
  • :active

In your .css, it is very important to define these elements in this order to avoid accidental overrides of style formatting.

:link

Select links in elements and change their formatting.

:visited

Select links in elements that have previously been activated and change their formatting.

:active

Select links in elements that are currently being activated (clicked on) and change their formatting.

:hover

Select links which are currently being touched by the cursor (but not being activated) and change their formatting.

Default settings for linking selectors

(html5)

  • :link {color: #0000EE;} (blue)
  • :visited {color: #551A8B;} (purple)
  • :hover {text-decoration: underline;}
  • :active {color: #EE0000;} (red)

Defaults look like:

:link

:visited

:hover

:active

But we can make our own!

:link{color: #E6005C;}

link

:visited{color: #00FFFF;}

hover{color: #FFFF00;}

hover

active{color: #FF0000;}

active

Privacy Safeguards:

Usually any text effects can be used.

However, to protect people's privacy, browsers restrict the effects used for :visited to color effects.

Use on Mobile Devices

:link, :visited, and :active work normally on mobile browsers.

:hover cannot be used on any mobile device with a touch screen because there is no cursor to hover over an element.

Use of the focus pseudo-selector is mostly for keyboard navagation of a site

This is not just limited to links but to input boxes and text as well

Here is what the CSS code would look like, and in action

a:focus { background: yellow; }

The target pseudo-selector is a powerful tool used not just to link to sites but to specific sections of a page

the pseudo-selector will jump to a point in the page where the hash mark in the url matches the I.D. of the selector

an example of the target pseudo-selector in action can be found here

Here is what that CSS code would look like

#voters:target {}

and the HTML

(section id="voters"> Content (/section>

The enabled and disabled pseudo-selectors are used to select enabled and disabled areas of input

Here is what that code would look like in CSS

input:enabled{color: #333;font-style: italic;} input:disabled{background: url(data:image/png;base64,iVBOR...) repeat;}

and in HTML

(label for="name">Enabled Input:(/label) (input type="text" autofocus) (label for="name">Disabled Input:(/label( (input type="text" disabled)

The checked pseudo selector effects check boxes in different states

Here is what that code looks like in CSS

input[type=checkbox] + label {color: #ccc;} input[type=checkbox]:checked + label {color: #0A0;}

and HTML

(input type="checkbox" id="ossm" name="ossm") (label for="ossm">change my color(/label)

The ":empty" pseudo-selector is used to apply styles to elements that contain no children.

Note: Comments do not count as children.

The syntax for the :empty pseudo-selector is:

 <element>:empty {style properties}

For example:

.box:empty {background: lime;}

HTML:

<div class="box"><!-- This is lime --></div>

<div class="box">This is red</div>

Empty CSS:

.box:empty {background: lime;}

Not empty CSS

.box {

background: red;

height: 200px;

width: 200px;}

This is red

The :first-child pseudo-selector will apply styling to the first child of it's parent.

CSS:

div :first-child

{ background-color: lime; }

HTML

<div>

<span>FIRST-CHILD</span>

<span>SECOND CHILD</span>

</div>

FIRST-CHILD SECOND CHILD

The :last-child pseudo-selector will apply styling to the last child of it's parent.

CSS:

div :last-child

{ background-color: lime; }

HTML

<div>

<span>FIRST-CHILD</span>

<span>LAST CHILD</span>

</div>

FIRST-CHILD LAST-CHILD

The :first-of-type pseudo-selector will apply styling to the first of each element type within the given element.

HTML:

div :first-of-type

{ background-color: lime; }

CSS

<div>

<span>FIRST</span>

<span>NOT FIRST</span>

<p>FIRST</p>

</div>

FIRST NOT FIRST FIRST

The :last-of-type pseudo-selector will apply styling to the last of each element type within the given element.

HTML:

div :last-of-type

{ background-color: lime; }

CSS

<div>

<span>NOT LAST</span>

<span>LAST</span>

<p>NOT LAST</p>

<p>LAST</p>

</div>

NOT LAST LAST

NOT LAST

LAST
